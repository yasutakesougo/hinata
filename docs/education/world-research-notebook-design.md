# 🧪 WRL-2「世界研究ノート（せかい攻略ノート）」設計ガイドライン

## 🧭 設計の核心
WRL-2「世界研究ノート」は、娘さんを説得したり管理したりするためのものではなく、**「世界の仕様（ルールや他者の性質）をパパ・ママと一緒にデバッグする」**ためのノートです。

実装にあたっては、UIとデータ構造の時点で**「子ども向けの見通し」**と**「親向けの背景情報（翻訳メモ）」**を厳格に分離します。これにより、子ども画面に認知的負荷の高い長文や、意図しない監視感を与える表現が露出するリスクを排除します。

---

## 📐 分離設計ルール

### 1. 子ども画面の表示原則
* 子ども用の表示領域には、親向けの解説や長文説明を表示しない。
* **`title`（子ども向け表示名）**、**`debugName`（親向け・内部名）**、**`childSummary`（みとおし）**、**`commands`（つかえるコマンド）** のみを表示し、シンプルかつ直感的な操作感にする。

### 2. `childSummary` の記述制約
* **文字数・文数**：1〜3文までとし、1文は短く区切る。
* **表記**：ひらがな中心（カタカナは適宜使用）。
* **トーン＆マナー**：
  * 「あなたが悪い」「あなたのルールが間違い」という自己否定感を抱かせる表現は一切禁止。
  * 「相手が一方的に悪い」という悪者化にも寄せない（世界の仕様・特性として説明する）。
  * 最後は必ず**「安心（あなたは間違っていない）」**か**「具体的な見通し」**で終わる。

### 3. `parentNote` の役割
* 親が子どもに「その場で口頭で翻訳して伝える」ための裏メモ（カンペ）。
* 認知心理学や発達特性の背景を記載し、親自身がフラットな視点（メンタライジング）を取り戻すための情報源とする。

---

## 📊 データスキーマ（TypeScript型定義の基本構想）

```typescript
export interface ResearchCard {
  id: string;
  title: string;            // 子ども向け：表示名（例：「やくそくが かわっちゃった日」）
  debugName?: string;       // 親向け・内部名（例：「やくそく上書きバグ」）
  childSummary: string;     // 子ども向け：1〜3文のひらがな要約（安心と見通し）
  commands: string[];       // 子ども向け：次に使えるアクションコマンド（ひらがな）
  parentNote: {
    detail: string;         // 親向け：想定される具体的な事象
    specification: string;  // 親向け：認知心理学や発達段階から見た「世界の仕様」
    memo?: string;          // 親向け：学校や療育で共有する際の視点
  };
}
```

## 🗂️ サンプルカード 3選

### 1. やくそくが かわっちゃった日
* **子ども向け表示名 (`title`)**：やくそくが かわっちゃった日
* **親向け・内部名 (`debugName`)**：やくそく上書きバグ
* **子ども向け要約 (`childSummary`)**：
  > やくそくを わすれちゃうことが あるよ。あなたの ルールが まちがい ということでは ないよ。こまったら、もういちど きいても だいじょうぶ。
* **つかえるコマンド (`commands`)**：
  * [ ] もういちど きいてみる
  * [ ] べつの あそびにする
  * [ ] パパ・ママに ほうこくする
* **親向け仕様 (`parentNote`)**：
  * **想定される事象**：「約束したのに、別の楽しい遊びに吸い込まれて行ってしまった」
  * **世界の仕様**：低学年や幼児の脳は、目の前に新しい魅力的な刺激が現れると、ワーキングメモリ内の「前の約束」が容易に上書きされてしまう。悪意があるわけではない。娘の「約束は守るべき」という認知は正しいことを保証しつつ、次のコミュニケーションコマンドへ誘導する。

### 2. じゅんばんが かわっちゃった日
* **子ども向け表示名 (`title`)**：じゅんばんが かわっちゃった日
* **親向け・内部名 (`debugName`)**：じゅんばんワープバグ
* **子ども向け要約 (`childSummary`)**：
  > ならんでいるのが みえなくて、まえに はいっちゃう人が いるよ。あなたが わるいわけでは ないよ。こまったら、おとなと いっしょに ならびなおそう。
* **つかえるコマンド (`commands`)**：
  * [ ] 「ならんでいるよ」と やさしくいう
  * [ ] パパやママの てを にぎる
  * [ ] すこし はなれて まつ
* **親向け仕様 (`parentNote`)**：
  * **想定される事象**：「列に並んでいたのに、横から割り込まれた」
  * **世界の仕様**：視野の狭さや、自分の目的物に対する衝動的な集中（自己中心性）により、物理的に「列があること」や「他者が先に待っていること」を認知できていないケースが多い。娘の被害感情を最小限に抑えるため、「相手には見えていなかった」という物理的視点（仕様）を伝え、大人が介入する隙間を作る。

### 3. こえが おおきくて つかれた日
* **子ども向け表示名 (`title`)**：こえが おおきくて つかれた日
* **親向け・内部名 (`debugName`)**：こえのボリューム大バグ
* **子ども向け要約 (`childSummary`)**：
  > うれしくて、こえが おおきくなっちゃう人が いるよ。あなたの みみが わるいわけでは ないよ。すこし はなれても だいじょうぶ。
* **つかえるコマンド (`commands`)**：
  * [ ] すこし はなれる
  * [ ] みみを やさしく ふさぐ
  * [ ] 「ちいさいこえで」と てのひらで サインする
* **親向け仕様 (`parentNote`)**：
  * **想定される事象**：「突然大きな声で話しかけられて、耳が痛くなったりびっくりして泣きそうになった」
  * **世界の仕様**：感情の興奮に伴って自身の発声音量を自己モニターできない（ブレーキが効かない）状態。聴覚過敏や驚愕反応の強い娘さんにとって脅威になるため、「怒っているわけではなく、感情の出力エラー」であることを説明し、物理的に距離を置いてセルフプロテクトするコマンドを提示する。
