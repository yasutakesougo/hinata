# 教育監修フィードバック報告書 (Advisor Feedback Report)

**作成日**: 2026年6月4日  
**監修者**: 教育・保育・発達支援 監修チーム  

本報告書は、開発中の幼児・低学年向け学習アプリ『さんすうクエスト 〜くだものキングダム〜』および『もりのひろば』の実機レビューに基づき、教育的効果、UX/UDL（ユニバーサルデザイン・フォー・ラーニング）適合性、および心理的安全性について評価し、製品化前に対応すべき改善提案をまとめたものです。

---

## 📋 全体評価

本アプリの**低刺激設計（感覚特性を持つお子様への配慮）**、**失敗を責めない心理的安全性**、および**UDLガイドラインへの適合度**は、既存の教育アプリと比較しても非常に高く評価できます。
具体的には、以下の点が教育現場および発達支援の視点から極めて優れています。
- ボス戦での白黒点滅（フラッシュ）などの強い視覚刺激を抑制する「うごきを とめる」オプションの実装。
- 誤答時でも「ちがうよ」「バツ（❌）」といった否定的な表現を避け、柔らかい声かけや「できた！」の割合による肯定的なフィードバックに寄せる設計。
- 親子が前向きに褒め合える温かみのある日本語表現（保護者レポートのリフレーミング）。

その上で、製品化に向けて学習効果を最大化するために、優先して対応すべき**3つの実装提言**を以下に示します。

## 🔍 各テーマの詳細と具体的な方針

### 1. 算数カリキュラム順序の再構成 (P1)
**現状の学習順序**:
`合成(5) ➔ ひきざん(5) ➔ 数の分解(10) ➔ 合成(10) ➔ ひきざん(10) ➔ 10づくり ➔ ボスMIX`

**監修からの提言と見直し理由**:
「ひきざん」という形式的な記号操作に入る前に、必ず**「数の分解（分ける力）」**を習得する順序に再構成することを強く推奨します。
ひきざんは、頭の中で「数を分ける力（数の合成・分解）」が育っていない段階で導入すると、指折りのカウントダウン（指で順番に戻り算をする方法）に過度に依存しやすくなり、数概念の発達を阻害する原因となります。

**再構成後のカリキュラム案**:
1. **5までの合成・分解**（Stage 1: 5までの合成 ➔ Stage 2: 5までの分解）
2. **10までの合成・分解**（Stage 3: 10までの合成 ➔ Stage 4: 10までの分解 ➔ Stage 5: 10づくり）
3. **形式的な記号操作**（Stage 6: 形式的なひきざん ➔ Stage 7: たしざん・ひきざんMIXボス）

---

### 2. 誤答時ヒントのスキャフォールディング (P1)
**監修からの提言**:
間違えた際に、単に「もういちど、よくかぞえてみよう！」と繰り返すだけでは、幼児がどこを直していいか分からず挫折しやすくなります。
間違えた回数（Attempt）に応じてヒントを段階的（スキャフォールディング）に提示する仕組みを導入してください。

- **1回目の誤答**: 自分で数え直すのを促す優しい声かけ（現状通り）。
- **2回目以降の誤答**: 具体的な数値を交えたヒント（例：「ひだりに 3びき いれると、みぎは 2ひき だよ」「あわせて 5こ になるものを えらんでね」など）を表示・発声する。

---

### 3. なぞり書きの判定方針整理 (P2)
**監修からの提言**:
文字の形を厳しく判定するのではなく、**「書き切った達成感」**を最優先してください。
- **軌跡のブレ・はみ出し**: 判定を甘く設定し、ブレていても書き切れば合格とする。
- **書き順と方向の支援**: 始点・なぞるべき方向・次の一画の表示をより明瞭に示す。
- **フィードバック**: 間違えた軌跡を描いたときに怒るのではなく、正しい方向になぞれたときだけリアルタイムに反応し、最後まで書けたら最大級に褒める。

---

### 4. 保護者レポートのプロセス承認拡張 (P2)
**監修からの提言**:
「正答率」を「できた！の割合」に置き換えた設計は、前向きな親子コミュニケーションのために非常に高く評価できます。
さらに、子どもの**非認知能力や主体的なプロセス（努力）**を保護者が承認できるよう、以下のプロセスメトリクスを追加することを推奨します。
- 「きょうは 〇分 がんばってとりくみました」
- 「まちがえても、もういっかい やってみる ちからが でました」
- 「おなじもんだいに 〇かい チャレンジしました」

---

### 5. もりのひろばの季節変化・探索要素 (P3)
**監修からの提言**:
幼児が継続してアプリを開きたくなるよう、ハブ画面である『もりのひろば』において、現実の季節に応じた背景変化（夏はヒマワリ、冬は雪など）や、タップするとどうぶつがリアクションするようなちょっとした探索要素を後続で追加していくと効果的です。
