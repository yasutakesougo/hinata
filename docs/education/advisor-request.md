# 教育監修依頼文（アドバイザー向け）

本ドキュメントは、幼児〜小学校低学年向け教育アプリ『さんすうクエスト 〜くだものキングダム〜』および『森の広場』の教育監修を専門家の方へご依頼する際にお渡しする依頼文・説明資料のテンプレートです。

---

## 1. アプリのご紹介と監修依頼についてのお願い

### ご挨拶
教育・保育・発達心理などの専門家の皆様

いつも子どもたちの健やかな成長と学びを支えてくださり、心より感謝申し上げます。

私たちは、5歳〜7歳（幼稚園・保育園の年長から小学校1年生）の子どもたちが、安心して、自分のペースで楽しく学べる教育アプリの開発を行っております。

現在、アプリの基本機能が完成し、実際の製品化・配信に向けたブラッシュアップを進めております。しかし、開発チームだけでは「本当にこの難易度で子どもが挫折せずに取り組めるか」「感覚特性を持つ子にとって負担にならないか」「保護者の方が安心して使わせられるか」といった点について、十分な検証が困難です。

そこで、現場での指導経験や専門的知見をお持ちの皆様に、本アプリの「教育監修・アドバイザー」としてお力添えをいただきたく、このたびご依頼を差し上げました。

子どもの知的好奇心を刺激しつつ、心身の健康と安心を守るためのアプリづくりに向けて、専門的な視点から忌憚のないご意見をいただけますと幸いです。

## 3. 私たちが大切にしている設計思想

本アプリは、一般的な「ゲーム要素で子どもを引きつけるアプリ」とは異なり、以下の**5つの安心・低刺激設計**を最優先に開発しています。

1. **過度な刺激（ハイクオリティだが低刺激）の排除**
   - パステル調の穏やかな配色を採用し、強い光や明滅、激しい揺れ（シェイク）などのエフェクトを抑えています。
   - 感覚過敏や注意力が散漫になりやすいお子様でも、落ち着いて画面に向き合える環境を目指しています。
2. **広告・ガチャ・課金誘導の完全な排除**
   - 子どもの誤操作を誘発するバナー広告や、射幸心を煽るガチャシステムは一切搭載していません。
   - 純粋な学びの達成感（動物が仲間になるなど）を報酬としています。
3. **時間制限やペナルティの排除**
   - 「早く答えないとゲームオーバー」「タイムアップ」といった、子どもを焦らせる・急かす設計を排除しています。
   - じっくり考えて自分で答えを導き出す時間を保障します。
4. **挫折させない・傷つけないフィードバック**
   - 間違えた場合でも「ブブー」という不快なエラー音や否定的な表現は使いません。
   - 「のせすぎだよ、すこしへらそう！」「もういちど、よくかぞえてみよう！」といったマイルドで具体的なヒントを音声とテキストで提示します。
5. **保護者との温かい連携**
   - 単に「勉強をさせた」で終わらせず、保護者向け画面（がんばりレポート）を通じて、お子様の「がんばりのプロセス（回答数や履歴）」を可視化します。家庭での前向きな声かけをサポートします。

## 4. 監修をお願いしたい範囲

以下の観点について、アプリの実機画面や仕様書を確認いただき、フィードバックをお願いしたいと考えております。

1. **年齢適合性（言葉と画面構成）**
   - 画面内の文字量、音声読み上げのスピード、漢字の有無などが5〜7歳に最適か。
2. **学習設計（難易度とステップ）**
   - 算数（合成 ➔ 10の補数 ➔ 引き算 ➔ 数の分解）の出題順や難易度の上がり方が自然か。
   - ひらがなのなぞり書きにおいて、判定基準やガイド矢印が適切か。
3. **声かけ・メッセージ（言葉の選定）**
   - 動物コンパニオン（相棒の動物）の発言や、正解・不正解時のメッセージが適切か。
   - 「もっとやろう！」といった過剰なプレイの煽りになっていないか。
4. **低刺激・発達特性への配慮**
   - アニメーションや効果音が感覚過敏のお子様にとって強すぎないか。
   - 「うごきを とめる」設定などのアクセシビリティ機能が有効に働くか。
5. **保護者向けの設計**
   - がんばりレポートの情報（回答率や履歴）が、親子のコミュニケーションに役立つ形になっているか。

## 5. 初回レビューにおける質問リスト

監修をスタートするにあたり、まずは以下の質問についてご意見をいただけますと幸いです。

### Q1. UIと言葉づかいについて
* アプリのタイトル画面から学習画面にかけて、5〜7歳のお子様が「大人のサポートなしで一人で進められる」言葉づかい・画面構成になっているでしょうか？
* 漢字の使用割合や、ボタンに添えられたひらがなのフォント・サイズに関して改善すべき点はありますでしょうか？

### Q2. 算数コンテンツのステップについて
* 「合成（たしざん）」「10づくり（たまごパック）」「ひきざん（のこりはいくつ）」「お部屋分け（数の分解）」というステップは、小学校算数へのスムーズな導入として適切な構成でしょうか？
* 途中の「ボス戦（3択問題）」は、子どもにとって適度な緊張感（達成感）になっているでしょうか。それとも負担が大きいでしょうか？

### Q3. 間違えた時のフィードバックと声かけについて
* 問題を間違えた際、音声と文字で「まだたりないよ、もっとのせてね！」「のせすぎだよ、すこしへらそう！」などのヒントを出しています。このフィードバックはお子様の再挑戦意欲を阻害しない設計になっているでしょうか？
* より良いヒントの出し方や、子どもを傷つけない代替表現があればご提案ください。

### Q4. 低刺激UXと発達特性への配慮について
* 画面のパステルカラーや、正解時の「星が飛び散るアニメーション」等は、感覚過敏や多動傾向のあるお子様にとって適切な刺激レベルでしょうか？
* Phase 3として「うごきを とめる（アニメーション軽減）」「コントラスト比の厳格化」「ドラッグ＆ドロップの代替タップ操作」を予定しています。これらの優先度や、他に備えるべき配慮（例: 音声合成の速度調整など）があれば教えてください。

### Q5. 保護者レポート（がんばりレポート）について
* 「がんばりレポート」に表示される情報（総回答数、正答率、学習履歴）は、保護者の方から見て分かりやすく、お子様を褒めるきっかけになるでしょうか？
* 「正答率が低くても傷つかない表現」や「家庭でのアドバイス方法の提示」など、保護者向け画面で追加すべきコンテンツがあればご提案ください。
