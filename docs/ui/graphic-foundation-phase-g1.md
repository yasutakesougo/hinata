# Phase G-1: ひなたグラフィック基盤整備 (Graphic Foundation)

本ドキュメントは、ひなたアプリにおける「絵本のように安心できる上質さ」と「触りたくなるUI（Lv.2〜3）」を実装するためのグラフィック基盤整備（Phase G-1）の設計・実装内容をまとめたものです。

---

## 🧭 設計の方向性
- **低刺激かつ高品質**: 派手なゲーム的アニメーションや原色、高コントラストすぎるエフェクトを避け、低彩度の優しいアースカラーとぬくもりのある木・紙の質感を基本とします。
- **安心感と直感性**: 成功時に過剰に喜ばない・失敗時に悲しまない演出に配慮。ボタンや操作対象は「押せる」と直感できるよう3D的な奥行き（底面シャドウと沈み込み）を付与しています。
- **UDL（ユニバーサルデザイン）対応の徹底**: パステル背景の上のテキストでもコントラスト比を **4.5:1以上**（濃い茶色や深緑）に保ち、Reduced Motion（うごきを止める）設定時は全画面のアニメーションを一括で安全に停止します。

---

## 🛠️ 主な実装内容

### 1. カラーパレットのCSS変数化（テーマ対応）
Tailwind 4 の `@theme` ディレクティブとCSS変数を連動させ、親要素のクラス（`theme-cream`, `theme-pink` 等）を切り替えるだけで背景、カード、テキスト、アクセントカラーが連動して切り替わる仕組みを導入しました。

- **デフォルト (クリーム) / `theme-cream`**:
  - 背景色: `#FFFEEB` / カード: `#FFFDF2` / 境界線: `#F6E6CA` / テキスト: `#4E3618`
- **さくら / `theme-pink`**:
  - 背景色: `#FFF0F5` / カード: `#FFFBFD` / 境界線: `#FCDCE3` / テキスト: `#66182C`
- **ラベンダー / `theme-purple`**:
  - 背景色: `#F3E8FF` / カード: `#FDFBFF` / 境界線: `#EBD5FF` / テキスト: `#4314A6`
- **オレンジ / `theme-orange`**:
  - 背景色: `#FFF5EB` / カード: `#FFFBF7` / 境界線: `#FFE6D6` / テキスト: `#5C1E0A`
- **そらいろ / `theme-sky`**:
  - 背景色: `#F0F9FF` / カード: `#FAFDFE` / 境界線: `#E0F2FE` / テキスト: `#034D72`

### 2. 共通UIコンポーネントスタイル
- **絵本風カード (`.hinata-card`)**: 緩やかな角丸 (`rounded-3xl`) と太めのボーダー、浮き上がりを感じる影 (`shadow-[0_8px_0_var(--hinata-border)]`) を設定。
- **木の案内板 (`.hinata-card-wood`)**: 気分決定やもり選択のモーダルで使用。温かみのあるブラウン (`#FAF0DD`) と太い木製枠線、ロープ装飾（CSSで表現）で構築。
- **3D押し下げボタン (`.hinata-btn-primary`, `.hinata-btn-secondary`, `.hinata-btn-wood`)**: 太い底面シャドウを持ち、アクティブ時に `translate-y-[2px]` しながら底面シャドウが縮む 3D アクションフィードバックを実装。

### 3. 森の絵本風パターン背景 (`.hinata-pattern-dots`)
全体の背景（`main` コンテナ）に、うっすらとしたドットパターンを透過させて重ねることで、のっぺりとした単調な塗りを防ぎ、絵本のような優しい手触り感を表現しました。

### 4. 算数活動の共通フレーム (`.hinata-activity-frame`)
4つの算数活動（おへやわけ、おやつわけ、たいこ、さくらんぼ）の外枠、戻るボタン、進捗スター（`StarProgress`）のレイアウトとビジュアルを共通化しました。
- **`StarProgress`**: 星アイコンを円で囲み、スタンプを押したような優しいビジュアルへ変更。
- **ともだちたいこ (`FriendTaikoScreen`)**: 太鼓ボタンの影の厚みをさらに強調し、叩いたときにリアルにおもちゃをトントンしているような操作感を向上させました。

### 5. data属性による一括 Reduced Motion 制御
アプリの設定「うごきを とめる」が ON のとき、最上部ラッパーに `data-reduced-motion="true"` 属性を付与します。
CSS側で `[data-reduced-motion="true"] * { animation: none !important; transition: none !important; }` を適用し、3択の浮遊、背景の落ち葉、動物の歩行、ボタンの跳ねなどのすべてのループ/トランジションを確実に一元停止します。
