# 実機プレイテスト記録テンプレート (Playtest Observation Checklist)

本ドキュメントは、幼児〜小学校低学年のお子様およびその保護者様に、実機（iPadやスマートフォン等）でアプリをテストプレイしてもらう際の観察・評価項目をまとめたテンプレートです。

開発チームやテスト実施者がこのチェックリストを印刷または手元に用意し、プレイ中の様子や対話から得られた気づきを記録して、**Phase 6D（プレイテスト反映）** の改善へと繋げます。

## 👦 お子様の観察チェックリスト

プレイ中のお子様の表情、手の動き、つぶやき、躊躇した場面などを観察し、以下に沿って記録します。

### 1. 今日のおすすめと意思決定 (自律性)
| 観察項目（チェックポイント） | 評価 (◯/△/✕) | 具体的な様子・つぶやき・課題点 |
| :--- | :---: | :--- |
| **迷いすぎずに選べているか**<br>「きょうは なにから はじめる？」の3択画面が表示された際、迷ったり困ったりせず、興味を持ってカードを選べているか？ | | |
| **「あとでえらぶ」を自然に使えるか**<br>その時に「何もしたくない」または「ホーム画面に行きたい」と感じたとき、画面外をタップする等でスムーズにスキップできているか？ | | |
| **さんすうクエストを選んだ時の表情**<br>「さんすうをする」を選択した際、プレッシャーを感じて嫌がったり、顔を曇らせたりしていないか？ | | |

### 2. クエストのプレイ体験 (学習・操作性)
| 観察項目（チェックポイント） | 評価 (◯/△/✕) | 具体的な様子・つぶやき・課題点 |
| :--- | :---: | :--- |
| **文章題の読み上げ (TTS) の効果**<br>問題開始時の「おはなし（文章題）」の読み上げをしっかり聞いて、問題の意味を把握できているか？ | | |
| **ひき算の ❌ 表示の視覚的理解**<br>ひき算問題において、半透明化と ❌ マークのついたくだものを見て、「引く（引いた後に残る）」という構造をイメージできているか？ | | |
| **ドラッグ＆タップの操作性**<br>お部屋分けなどで、ドラッグ操作だけでなく、調整ボタン（＋/－）やタップ移動を使ってストレスなく操作できているか？（タッチターゲットは適切か） | | |

### 3. レジリエンスとサポート (安心感)
| 観察項目（チェックポイント） | 評価 (◯/△/✕) | 具体的な様子・つぶやき・課題点 |
| :--- | :---: | :--- |
| **間違えた後の解き直しの姿勢**<br>間違えたときに、不快な音やエラー表示に怯えることなく、自然に「もう一度やってみよう」と思えているか？ | | |
| **ヒント表示時の安心感**<br>2回間違えた際に出るヒント（テキスト・音声）を、責められていると感じず、ヒントを手がかりにして解き直せているか？ | | |
| **刺激量の適切さ**<br>バトル演出（さんすうキング）やクリア時などのアニメーションや点滅、BGMが、興奮しすぎさせたり、逆に感覚的にうるさく感じさせたりしていないか？ | | |

## 👩 保護者様のヒアリング・観察チェックリスト

プレイ後、またはプレイ中の保護者様の反応から、レポート画面や機能の受け取られ方を評価します。

### 1. がんばりレポートの受容とフィードバック
| 観察項目（チェックポイント） | 評価 (◯/△/✕) | 具体的な様子・保護者様のコメント |
| :--- | :---: | :--- |
| **「声かけの材料」になっているか**<br>レポートが単なる「できた問題の点数チェック」ではなく、「今日は自分で選べたね」「ヒントを見て諦めなかったね」といった声かけの具体的な手がかりとして使えているか？ | | |
| **「ヒントを使った」ことへの肯定的な評価**<br>「ヒントを〇回使用しました」という情報を、保護者様が「自力で解けなかった悪いこと」ではなく「サポートを活かせた良いプロセス」として前向きに捉えられているか？ | | |
| **「再挑戦」への肯定的な評価**<br>「あきらめないちから」などの表現から、間違えたこと自体ではなく、解き直そうとしたしなやかさ（レジリエンス）を自然に褒められているか？ | | |

### 2. 安心感とUI設計
| 観察項目（チェックポイント） | 評価 (◯/△/✕) | 具体的な様子・保護者様のコメント |
| :--- | :---: | :--- |
| **何もしていない日の配慮**<br>開いただけで終わった日（アクティブ時間が極めて短い日）に、「今日のがんばり」のテキストが無理に盛りすぎた褒め方にならず、寄り添う形で自然に見えるか？ | | |
| **プレッシャーのなさ**<br>レポート画面を万が一お子様が隣で一緒に覗き込んで見た場合でも、焦りやプレッシャーを与えない優しいトーンが保たれているか？ | | |
| **設定の使いやすさ**<br>「うごきを とめる（Reduced Motion）」や「よみあげスピード（ふつう/ゆっくり）」の設定を、保護者が直感的に理解し、調整できているか？ | | |
