# さんすうクエスト 〜くだものキングダム〜 🍎👑

小学校1年生向けにデザインされた、体感型・音感型の算数ゲームアプリです。合成（たしざん）、10づくり（補数）、ひきざん（残数）を、くだものや可愛い動物たちと一緒に楽しく学べます。

---

## 🎮 ゲームの特徴
1. **ごうせい（たしざん）**: 左右の皿にあるくだものを「がったい」させて、合計を数えます。
2. **10づくり（たまごパック）**: 10マスのたまごパックをぴったり満たすために、あといくつくだものをのせるかを考えます。
3. **ひきざん（のこりはいくつ）**: くいしんぼうな動物（ハムスターやオウム）がくだものを食べてしまい、残った数を数えます。
4. **なかまずかん**: ステージをクリアすると仲間になる動物たちを集めるコレクション機能。
5. **がんばりレポート（保護者向け）**: 計算の回答数、正答率、履歴を確認できるダッシュボード。クラウドでのセーブデータ同期に対応。

---

## 🛠️ 技術スタック
- **フロントエンド**: React 19, TypeScript, Vite
- **スタイリング**: Tailwind CSS v4 (CSS-first config)
- **インフラ**: Firebase Auth (匿名ログイン / メールリンク同期), Cloud Firestore
- **オーディオ**: Web Audio API (動的コード合成BGM & 音響効果), Web Speech Synthesis API (日本語音声ガイド)

## 📋 配布前最終安全チェックリスト

実機でのテスト配信を行う前に、以下の2つのテスト環境において動作確認を行ってください。

### A. LAN表示・操作性確認（ローカルHTTP環境）
> **テスト方法**: `npm run dev -- --host` を実行し、LAN内IP（`http://192.168.x.x:5173`）経由で同一Wi-Fi上の実機ブラウザから開いて確認します。

- [ ] **表示・レイアウト確認**: 異なるデバイス（iPad, iPhone, Android）で、10マスの枠（たまごパックや合成盤面）がはみ出さず、くだものがグリッド内に綺麗に収まっていること。果物同士が重なり合っていないこと。
- [ ] **操作性（タッチターゲット）**: ヘッダーの戻るボタン、音量ボタン、答えの数字キーなどが、子どもの指でも押しやすいサイズ（44px × 44px以上、PWA要件としては48px以上）に保たれていること。
- [ ] **音量一括ミュート**: 音をOFFに切り替えた際、BGM・タップ音・正解音・音声読み上げが全て無音になること。
- [ ] **子どもの連打対策**: 合成ボタンや回答ボタン、クリア画面の「つぎへ」ボタンなどを連打した際に、問題がスキップされたり、ボス戦のHPが多重に減るなどのバグが起きないこと。

### B. PWA・オフライン起動確認（HTTPSデプロイ環境）
> **テスト方法**: `npm run build` 後、`npx firebase deploy --only hosting` を行ってHTTPSの環境にデプロイし、そのセキュアなURLから起動して確認します。（※Service WorkerはセキュアコンテキストであるHTTPSでのみ登録可能なため）

- [ ] **PWAインストール**:
  - **iOS Safari**: 共有ボタン ➔ **「ホーム画面に追加」** が行えること。アプリアイコン（王冠りんご）が正常に表示されること。
  - **Android Chrome**: メニュー ➔ **「アプリをインストール」** が行えること。
- [ ] **全画面化確認**: ホーム画面に追加したアイコンから起動した際、アドレスバーやブラウザの下部メニューが表示されず、フルスクリーンで起動すること。
- [ ] **機内モード起動**: 端末を機内モード（完全にオフライン）にし、ホーム画面のアイコンから起動した際、3秒以内にアプリが起動してタイトル画面が表示されること。
- [ ] **オフラインでの保存**: オフラインのままプレイし、ステージをクリアして動物をアンロック。アプリを一度完全に終了して再起動後、進捗が消えずに残っていること。
- [ ] **オンライン復帰と同期**: インターネット接続をONに戻し、保護者メニューを開いた際に、保存状態が `🟡 この端末に保存中` または `🔵 つうしんまち` から `🟢 ほぞんずみ（クラウド同期完了）` に自動で遷移すること。

## 📱 PWA（ホーム画面追加）インストール手順 ＆ 端末別確認項目

本作は PWA（Progressive Web App）に対応しており、端末のホーム画面に追加することで、インターネット接続がないオフライン環境下でもネイティブアプリのように起動・プレイ可能です。

### 1. インストール手順

#### 📲 iPad Safari / iPhone Safari
1. **Safari** で本アプリのURLを開きます。
2. 画面下部（iPhone）または上部（iPad）の **共有ボタン**（四角から上矢印が飛び出したアイコン）をタップします。
3. メニューをスクロールし、**「ホーム画面に追加」** を選択します。
4. アプリ名を確認し、右上にある **「追加」** をタップすると、ホーム画面にかわいい「りんごのアイコン」が登録されます。

#### 🤖 Android Chrome
1. **Google Chrome** で本アプリのURLを開きます。
2. 右上の **メニューボタン**（縦の3点リーダー）をタップします。
3. **「アプリをインストール」** または **「ホーム画面に追加」** を選択します。
4. 画面の指示に従ってインストールを許可すると、ホーム画面にアプリアイコンが追加されます。

---

### 2. ブラウザ・端末別最終確認項目 (Checklist)

各端末の固有ブラウザ仕様に起因する不具合がないか、以下の項目を確認します。

#### 🍎 iPad / iPhone Safari (iOS / iPadOS)
- [ ] **ホーム追加時の全画面化**: ホーム画面からアイコンを起動した際に、ブラウザのアドレスバーや下部メニューバーが表示されず、フルスクリーンで起動すること。
- [ ] **サウンドの初期アンブロック**: 起動時に音が鳴らない場合、「ぼうけんをはじめる」またはスピーカーアイコンを1回タップした段階で、BGMと効果音の再生が正常にアンブロック（復帰）すること。
- [ ] **音声合成の言語**: 設問の読み上げが、おかしな英語風発音ではなく、明瞭な日本語（ja-JP）で行われていること。
- [ ] **ピンチイン・アウト禁止**: 子供が操作中に画面を誤って2本指でズーム（ピンチ）しようとした際、画面が拡大・縮小せず固定されていること。

#### 🤖 Android Chrome (Mobile Chrome)
- [ ] **アドレスバー自動非表示**: ホーム追加アイコンから起動した際に、アドレスバーが非表示になり、ゲーム画面が画面一杯に表示されること。
- [ ] **スリープ防止**: プレイ中に一定時間操作がない場合でも、読み上げやBGM演奏中に画面が急に暗くなったり自動ロックされないこと。
- [ ] **バックボタン（戻るキー）対策**: デバイス下部の「戻る」物理ボタン／ジェスチャーを行った際、ブラウザ離脱やタイトル画面の初期化が起きず、マップ画面へ戻る等の適切なゲーム制御内にとどまること。
